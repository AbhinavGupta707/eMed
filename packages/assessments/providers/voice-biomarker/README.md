# Local voice-biomarker provider

This provider is a research-only engineering implementation for the synthetic HomeRounds hackathon prototype. It is not clinically validated, does not diagnose or predict a condition, and must not be used with real patient data. Its numeric features can support a personal repeated-baseline demonstration only; they are not disease thresholds and cannot set urgency, select an action, or override the deterministic red-flag, protocol, quality, action, concurrency, or idempotency authorities.

The station requires explicit consent for a separate 6–8 second sustained-vowel capture. It does not analyze the ElevenLabs conversation. The browser requests a mono microphone stream with echo cancellation, noise suppression, and automatic gain control disabled where supported. AudioWorklet is preferred; the browser adapter includes a ScriptProcessor compatibility fallback for Safari implementations where the worklet path is unavailable. A secure context, `getUserMedia`, and Web Audio are required. Permission-query support is optional because some Safari versions expose microphone permission only when `getUserMedia` is called.

PCM exists only in memory. Worklet, compatibility, and provider-owned buffers are bounded and cleared or released during success, retry, cancellation, error, page interruption, and disposal. Tracks stop and the audio context closes after every attempt. No PCM, audio object, recording reference, transcript, provider payload, prompt, token, or secret is uploaded, logged, persisted, or returned. A passing fact always has `researchOnly: true` and `rawMediaRef: null`; retry, fail, and unavailable results contain no feature fact.

The dependency-free deterministic analysis uses windowed normalized autocorrelation and gates duration, clipping, harmonic-residual SNR, voiced fraction, and pitch stability before deriving median fundamental frequency, semitone pitch variability, local jitter, local shimmer, harmonic-to-noise ratio, and phonation duration. Reference fixtures document software tolerances only. Results remain sensitive to microphone response and placement, room noise and reverberation, sample rate, browser processing, language and vowel choice, loudness, age, hydration, and extraction method. Bluetooth routing, backgrounding, interruption, permission re-grant, physical iPhone Safari, and cross-device repeatability remain separate human/device evidence gates.
