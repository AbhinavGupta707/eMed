## Evidence classes

| Class | Meaning                                                                                                                                    |
| ----- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| `AD`  | Automated deterministic evidence: unit, contract, integration, package, configuration, or build checks using fixtures/fakes as documented. |
| `AB`  | Automated browser evidence: Playwright-controlled browser and viewport evidence; not a physical device unless explicitly stated.           |
| `OL`  | Observed local-runtime evidence: a real local process/database/browser inspection recorded by the Checkpoint 4 orchestrator.               |
| `DR`  | Document/configuration review: a checked-in control or procedure exists; this is not proof that an external platform executed it.          |
| `PX`  | Pending external/human evidence: account, provider, physical device, advisory service, or qualified review not observed on the base.       |
